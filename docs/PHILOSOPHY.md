# MemFlow 设计哲学

## 使命

MemFlow 旨在构建一个**Agent 原生的工作流执行引擎**，通过融合大型语言模型（LLM）与结构化工作流，实现：

- **极省 Token**：通过记忆驱动的优化，持续降低 AI 调用成本
- **自我进化**：基于用户反馈自动优化提示词和工作流
- **开箱即用**：兼容 n8n 生态，降低迁移成本

> 我们不追求替代 n8n，而是构建 n8n 无法实现的 Agent 增强层。

---

## 核心原则

### 1. 工作流即代码

一切工作流定义必须：
- 可版本控制（Git）
- 可审计（变更历史）
- 可测试（奇偶性测试套件）

```yaml
# 声明式工作流示例
name: Fetch GitHub Zen
steps:
  - id: request
    http:
      method: GET
      url: https://api.github.com/zen
```

### 2. 记忆驱动

跨会话学习是 MemFlow 的核心差异化：

- **用户偏好记忆**：自动提取用户偏好（如输出格式）
- **错误模式记忆**：记录常见的失败模式，避免重复
- **工作流优化**：基于用户修改历史优化 AI 提示词

```
用户修改 → Diff 记录 → 模式分析 → 提示优化
```

### 3. 默认安全

- **沙箱执行**：WASM/JS 插件运行在隔离环境
- **确认机制**：高危操作（如数据库删除）需用户确认
- **最小权限**：API Key 权限分级（只读/编辑/管理）

### 4. 可观测性优先

- 内置 Prometheus 指标
- 结构化日志（JSON）
- OpenTelemetry 兼容

## 架构边界

### 执行引擎的独立性

```rust
// 执行引擎不依赖特定 Agent 实现
pub struct Executor {
    pub env: Environment,
    // 通用指令集，不绑定具体 LLM
}
```

- 可独立运行：不受 Agent 服务状态影响
- 支持热重载：修改工作流无需重启

### 节点扩展机制

```
┌─────────────────────────────────────┐
│           插件系统                   │
├─────────────────────────────────────┤
│  WASM 插件    │  HTTP 调用  │ JS 脚本 │
│  (高性能沙箱)  │  (外部 API)  │ (快速原型)│
└─────────────────────────────────────┘
```

### 学习引擎与主路径解耦

```
┌──────────────┐     ┌──────────────┐
│  执行引擎    │ ──▶ │  学习引擎     │
│  (热路径)    │     │  (异步优化)   │
└──────────────┘     └──────────────┘
       │                    │
       ▼                    ▼
   工作流执行           Diff 分析
                       Prompt 优化
```

## 技术取舍

### 为什么选择 Rust？

- **性能**：接近 C 的执行效率，适合高并发场景
- **内存安全**：消除 GC 暂停，确保可预测延迟
- **WebAssembly**：支持插件化扩展

### 为什么使用 n8n JSON？

- **生态兼容**：可直接导入现有 n8n 工作流
- **社区丰富**：500+ 节点，1000+ 模板
- **学习曲线低**：用户已熟悉

### 为什么不是完全重写 n8n？

n8n 是优秀的低代码平台，但：
- 面向非技术用户，不适合 AI Agent 编程
- 工作流定义与执行强耦合
- 缺乏记忆和自我进化能力

MemFlow 定位：**n8n 的 Agent 增强层**，而非替代品。

## 贡献指南

### 代码风格

- 使用 `rustfmt` 格式化
- 遵循 Rust 命名约定（snake_case 函数，Pascal_case 类型）
- 所有公共 API 必须有文档注释

### 测试要求

- 单元测试覆盖核心逻辑
- 集成测试验证端到端流程
- 奇偶性测试防止回归

### 提交规范

使用 Lore Commit Protocol：
- 第一行：为什么改（非是什么）
- 正文：上下文和约束
- trailers: Constraint, Rejected, Tested 等
