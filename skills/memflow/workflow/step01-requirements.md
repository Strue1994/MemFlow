# 步骤 1: 需求收集

当用户请求创建工作流时，首先需要明确：

## 触发器类型
- 定时 (Schedule): "每天", "每小时", cron 表达式
- Webhook: "接收请求", "回调"
- 表单 (Form): "用户提交", "表单"
- 手动: "手动触发"

## 数据源
- 外部 API (HTTP)
- 数据库
- 文件
- 消息队列

## 目标动作
- 发送通知 (飞书, Slack, Telegram, Email)
- 写入数据库
- 调用外部 API
- 发送消息

## 示例对话

```
用户: 创建一个每天早上抓取 RSS 的工作流

助手: 明白了。请确认以下需求：

1. 触发方式: 定时 (每天早上)
2. 数据源: RSS 源
3. 目标: 飞书/邮件推送

是否正确？如有其他要求请告知。
```
