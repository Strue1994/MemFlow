# 任务 CC2-04：计划模式

## 目标
在真正执行工作流之前，先生成详细的执行计划（包括每个节点的输入输出预览），用户确认后再实际执行。借鉴 Claude Code 的 `plan` 权限模式。

## 文件
- `executor/src/dry_run.rs`（新建，模拟执行）
- `compiler/src/plan_generator.rs`（生成执行计划）

## 具体要求

### 1. 计划生成
- 用户发送请求时，若启用计划模式（`plan_mode=true`），Agent 不直接执行，而是调用执行引擎的 `dry_run` 方法。
- `dry_run` 遍历工作流图，对每个节点：
  - 如果输入参数已确定，则模拟执行（可调用节点的 `dry_run` 方法返回预期输出格式）。
  - 如果参数未知，标记为"待定"。
- 输出计划 JSON：节点顺序、预期输入/输出、可能的风险。

### 2. 计划展示
- Web UI 或 CLI 以树形或流程图展示计划。
- 用户可修改参数、跳过节点、插入确认点。

### 3. 确认执行
- 用户确认后，Agent 以正常模式执行工作流。
- 执行过程可与计划对比，发现偏差时提示用户。

### 4. 安全应用
- 高危节点（如数据库删除）在计划中高亮，要求用户明确确认。

## 验收标准
- 复杂工作流（10+ 节点）的计划生成时间 < 2 秒。
- 计划准确率（与实际执行结果对比）> 95%。
- 用户确认后执行无误。
