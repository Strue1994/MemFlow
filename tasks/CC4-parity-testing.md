# 任务 CC4：奇偶性测试套件

## 目标
建立一套"黄金输入输出"集，用于验证 MemFlow 执行引擎和编译器的行为一致性，防止重构引入回归。

## 文件
- `tests/parity/`（新建目录）
- `tests/parity/golden/`（存放黄金 JSON 文件）
- `tests/parity/test_runner.rs`（Rust 测试运行器）
- `tests/parity/workflow_generator.rs`（生成随机工作流）
- `scripts/update_golden.sh`（更新黄金文件）

## 具体要求

### 1. 黄金集格式
- 每个测试用例包含：
  - 工作流定义（n8n JSON）
  - 一组输入参数（JSON）
  - 预期输出（JSON）或执行过程中产生的中间值快照
- 示例文件：`tests/parity/golden/http_request.json`

### 2. 测试运行器
- 读取所有黄金文件，对每个用例：
  - 编译工作流为 IR。
  - 执行工作流，捕获输出。
  - 对比输出与预期（深度相等，允许浮点误差 1e-6）。
- 若有差异，生成 diff 报告并标记测试失败。

### 3. 随机工作流生成器
- 用于扩充测试覆盖：随机生成合法的工作流 DAG（限制节点数 ≤ 10）。
- 执行后自动记录输出作为新的黄金用例（需人工审核后加入）。

### 4. 集成到 CI
- 在 GitHub Actions 中运行奇偶性测试套件。
- 任何 PR 若导致黄金集差异，自动阻止合并并输出差异报告。

## 验收标准
- 黄金集至少包含 50 个不同结构的用例。
- 随机生成器能生成 1000 个不同工作流且无崩溃。
- CI 执行时间 < 5 分钟。
