# 任务 CC5：编写 PHILOSOPHY.md 与设计原则文档

## 目标
撰写一份清晰的 MemFlow 设计哲学文档，阐明项目的核心原则、架构边界和长期演进方向，为贡献者和用户提供决策依据。

## 文件
- `docs/PHILOSOPHY.md`（新建）
- `README.md`（增加指向哲学文档的链接）

## 具体要求

### 1. 内容大纲
- **使命**：为什么创建 MemFlow？（融合 Agent + 工作流，极省 token，自我进化）
- **核心原则**：
  - 工作流即代码：一切工作流定义可版本控制、可审计。
  - 记忆驱动：跨会话学习是核心差异化。
  - 默认安全：沙箱、确认机制、权限最小化。
  - 可观测性优先：指标、日志、追踪内置。
- **架构边界**：
  - 执行引擎的独立性（不依赖特定 Agent 实现）
  - 节点扩展机制（插件化）
  - 学习引擎与主执行路径解耦
- **技术取舍**：
  - 为什么选择 Rust？（性能、内存安全）
  - 为什么使用 n8n JSON 作为交换格式？（生态兼容）
  - 为什么不是完全重写 n8n？（聚焦 Agent 优化）
- **演进路线**：当前状态 → 短期目标（UI、更多节点）→ 长期愿景（分布式、更强学习）

### 2. 受众
- 新贡献者：理解代码组织方式和设计意图。
- 用户：了解系统能力和限制。
- 决策者：评估 MemFlow 是否适合其场景。

### 3. 风格
- 清晰、简洁、避免营销用语。
- 使用列表、图示（Mermaid）辅助说明。

## 验收标准
- 文档被所有核心开发者审阅并认可。
- 外部读者（如潜在用户）阅读后能准确复述项目核心价值。
- 文档中引用的所有原则在代码中有实际体现（例如"默认安全"对应任务 C3 的实现）。
