# 任务 E6：本地模型微调闭环

## 目标
利用用户反馈（设计修改、接受/拒绝）持续微调本地代码模型（如 OpenCode），使其生成更符合用户习惯的工作流。

## 文件
- `learning-engine/src/fine_tune_loop.rs`（新建）
- `scripts/fine_tune.sh`（调用微调脚本）
- `agent-service/src/feedback_collector.ts`（收集反馈）

## 具体要求

### 1. 反馈收集
- 当用户手动修改 AI 生成的工作流时，记录 (原始描述, 原始 JSON, 修改后 JSON)。
- 当用户接受或拒绝建议时，记录正负样本。

### 2. 微调触发
- 每积累 500 个反馈样本，自动触发一次微调。
- 使用 QLoRA 对基础模型（如 DeepSeek-Coder-6.7B）微调。

### 3. 模型版本管理
- 微调后的模型保存为新版本（带时间戳）。
- A/B 测试新模型：分配 10% 流量，若生成的工作流修改率降低 >10%，则全量替换。

### 4. 成本控制
- 微调使用竞价实例，避免影响主服务。

## 验收标准
- 经过 2000 个反馈样本后，模型生成的工作流首次修改率降低 30%。
- 微调过程不影响在线生成服务。
