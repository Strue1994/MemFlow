# 任务 S-5：自适应工作流推荐引擎

## 目标
基于用户历史执行行为，自动推荐可能有用或需要定期执行的工作流。

## 文件
- `learning-engine/src/recommender.rs`（新建）
- `learning-engine/src/sequence_miner.rs`（序列模式挖掘）
- `agent-service/src/recommendation_api.ts`（提供推荐接口）
- `web-ui/src/components/RecommendationPanel.tsx`

## 具体要求

### 1. 行为日志收集
- 记录每个用户执行工作流的时间戳、工作流 ID、执行结果（成功/失败）。
- 存储到 ClickHouse 表 `user_workflow_actions`。

### 2. 序列模式挖掘
- 每小时运行一次离线挖掘任务，使用 PrefixSpan 算法找出频繁序列（如 [A, B] 经常在 5 分钟内连续执行）。
- 将挖掘出的关联规则存入 Redis（TTL 24 小时）。

### 3. 实时推荐
- 当用户执行一个工作流后，查询关联规则，推荐下一个可能的工作流。
- 规则示例：`执行 "数据备份" 后，有 80% 概率执行 "发送报告"`。
- 推荐以非侵入方式展示（Web UI 侧边栏或聊天中提示）。

### 4. 周期性推荐
- 检测用户是否在固定时间（如每周一 9:00）执行同一工作流，若连续 3 周如此，则主动询问"是否要设置定时任务？"。
- 用户同意后，自动创建 Cron 触发器。

### 5. 个性化排序
- 使用协同过滤（基于用户相似度）为每个用户生成个性化推荐列表。
- 推荐理由可解释（如"因为您昨天执行了此工作流"）。

## 验收标准
- 用户连续执行工作流 A 和 B 多次后，下次执行 A 时推荐 B。
- 周期性工作流识别准确，主动询问。
- Web UI 中推荐面板显示合理建议。
